# Book Pipeline

Цей проєкт призначений для автоматизованої обробки великих текстів (до сотень тисяч символів) з подальшою адаптацією, переписуванням і генерацією нового художнього твору.

Основна ідея — розбити процес на етапи:

1. Аналіз оригіналу
2. Побудова структури (скелету)
3. Генерація нового тексту по сценах
4. Контроль якості через додаткові агенти

---

## 📁 Структура проєкту

```
Book_<назва>/
│
├── data/
│   ├── original/        # Оригінальний текст
│   ├── chunks/          # Розбиті частини (за потреби)
│   └── processed/
│       ├── bible.json   # Паспорт стилю
│       └── skeleton.json# Список сцен
│
├── prompts/
│   ├── bible.txt        # Промт для аналізу стилю
│   ├── skeleton.txt     # Промт для розбиття на сцени
│   ├── writer.txt       # Промт для генерації тексту
│   ├── editor.txt       # Промт для редагування
│   └── censor.txt       # Промт для перевірки якості
│
├── flows/
│   ├── build_bible.py   # Побудова стилю
│   ├── build_skeleton.py# Побудова структури
│   └── write_book.py    # Основний цикл генерації
│
├── state/
│   ├── memory.json      # Коротка пам’ять (summary)
│   └── characters.json  # Дані про персонажів
│
├── output/
│   ├── scenes/          # Згенеровані сцени
│   └── final.txt        # Фінальний текст
│
└── main.py              # Точка входу
```

---

## 🔻 Опис компонентів

### data/

Містить усі вхідні та проміжні дані.

* `original/` — повний текст книги
* `chunks/` — частини тексту для обробки
* `processed/` — результати аналізу

---

### prompts/

Тут зберігаються всі інструкції для моделей.

Це дозволяє:

* легко змінювати поведінку системи
* тестувати різні стилі
* не лізти в код

---

### flows/

Логіка пайплайну.

* `build_bible.py` — витягує стиль
* `build_skeleton.py` — будує сюжет
* `write_book.py` — генерує текст по сценах

---

### state/

Зберігає стан між ітераціями.

* пам’ять про попередні сцени
* узгодженість персонажів

---

### output/

Фінальні результати.

* кожна сцена окремо
* повністю зібраний текст

---

## 🔻 Логіка роботи

### 1. Аналіз (перший прохід)

* обробка великого тексту
* створення:

  * `bible.json`
  * `skeleton.json`

---

### 2. Генерація

Для кожної сцени:

1. Writer — пише текст
2. Editor — покращує стиль
3. Censor — перевіряє якість
4. Умова — повтор або далі

---

### 3. Збірка

* всі сцени об’єднуються
* формується фінальний файл

---

## 🔻 Принципи

* не працювати з усім текстом одразу
* використовувати сцени як базову одиницю
* тримати стиль окремо від сюжету
* використовувати коротку пам’ять, а не повний контекст

---

## 🔻 Призначення

Проєкт підходить для:

* адаптації текстів
* генерації художніх творів
* експериментів зі стилем
* побудови multi-agent систем

---

## 🔻 Важливо

* без “біблії” стиль розвалюється
* без “скелету” сюжет пливе
* без циклу перевірки якість падає

---

## 🔻 Рекомендації

* працювати великими блоками на етапі аналізу
* не змішувати етапи
* тестувати кожен flow окремо

---

Готова структура дозволяє масштабувати процес і уникнути втрати якості навіть на великих текстах.
